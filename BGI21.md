## 3 Optimized Distributed Comparison Function

比较函数秘密共享是点函数秘密共享的基础上需要计算额外的信息，因此对于随机生成器，其需要多生成额外的比特位。

对于点函数共享来说，其特殊路径上的点满足$s||1$，而对于所有不在特殊路径上的点则满足$0||0$，为了构造满足这一条件的树，并将其分发给两方进行计算，需要进行一些处理。

具体来说：

+ 对于生成方

1. 规定每一节点的形式为$s||t$，其中$s$表示长度为$n$的比特串，$t$表示长度为1的比特位。
2. 对于两方，各随机取一个种子$s$生成一代表特定输入的子树，子树中各节点之和可以理解一颗新的树，其满足特殊路径上的点满足$s||1$，而对于所有不在特殊路径上的点则满足$0||0$
3. 在每一层根据输入的不同，利用矫正词修改节点的$s||1$，使其满足构造树规定的值。对于生成方来说，它拥有两棵树的种子，所以 他可以根据两棵树的信息构造矫正词。（此为关键，具体下文进行介绍
4. 每一层都有一个矫正词，需要记录、其中矫正词是公共的。
5. 到达最后一层时，两棵树只有唯一节点互不相同，即用这个两个节点的$s$值作为掩蔽值对$\beta$进行秘密共享。只有两个子树都达到这以节点才能解密。



+ 对于计算方

1. 计算方能获得种子和矫正词组成的密钥。
2. 计算根据自己的输入选择节点进行下一层的扩展并用矫正词进行校正
3. 最后得到最后一层的种子，即$s||t$
4. 除了特殊路径，两棵子树的$s||t$ 相同，故其和为0。



以上即为点函数秘密共享的构造思路，如何在此基础上进行改造形成比较函数秘密共享的方案呢？

因比较函数具有一般性，故这里我们先只考虑小于函数。

在点函数中，我们规定每节点的形式为$s||t$，而在比较函数中，我们规定每个节点的形式为$s||v||t$。其中$s$，$t$仍遵循点函数中的规定，即如果其在特殊路径上，满足两子树之和为$s||1$，其余节点为$0||0$。

##  4.Public Intervals and Multiple Interval Containments

区间包含函数定义为：
$$
g_{IC} =\{ g_{IC,n,p,q}:U_N \rightarrow U_N \}   \mathbb{G}_{in}
$$

其中$g_{IC,n,p,q}:U_N \rightarrow U_N$ 表示为：
$$
g_{IC,n,p,q}(x)= \left\{ \begin{aligned} x =1 \\ x=1\end{aligned} \right
$$


### 1.1 FSS gate for $[p,q]$ using FSS scheme for $f^<_{(N-1)+r_in,1}$

由于需要掩蔽真实值，故需引入随机数$r$,故原本的区间包含需要进行转换，并在环上进行，即：$(p + r_{in}) \; mod \; N < (x + r_{in}) \; mod \; N < (q + r_{in}) \; mod \; N$ 

此时将会出现一个问题，原本的不等式在加上$r$后可能产生回绕现象，现在分情况讨论如下：

1. 当$(p + r_{in}) <N$ 且  $ (q + r_{in}) < N$ ，此时都不发生回绕，原不等式成立
2. 当$(p + r_{in}) <N$ 且  $ (q + r_{in}) > N$，即只有$q$发生了回绕，此时 $ (q + r_{in}) \; mod \; N < (p + r_{in}) \; mod \; N $ ,原不等式不成立
3. 当$(p + r_{in}) >N$ 且  $ (q + r_{in}) > N$，即$p,q$都产生了回绕，此时 $ (p + r_{in}) \; mod \; N < (q + r_{in}) \; mod \; N $原不等式成立

由此可见，仅有情况2时需要特殊处理。

首先，我们令$P(x)$表示$p < x < q$，并且用$1\{p< x < q\}$表示当$\{\}$内为真时输出位0，反之则为0。则$P(x)$可以等价为：
$$
P(x) = 1\{\widetilde{x} \leqslant \widetilde{q}\} - 1\{\widetilde{x} < \widetilde{p}\} + 1\{\widetilde{p} > \widetilde{q}\}
$$
